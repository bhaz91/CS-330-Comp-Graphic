# CS-330-Comp-Graphic

Throughout this course, I learned that designing software begins with understanding the requirements and creating a plan before writing code. For my project, I started by analyzing the reference image and identifying which basic shapes could be used to recreate the objects in a 3D environment. This process helped me develop stronger design skills related to visualization, object decomposition, and scene organization. I followed an iterative design process in which I planned, implemented, tested, and refined each component of the scene. These design tactics can be applied to future projects because they encourage careful planning and continuous improvement.

My approach to developing programs involves breaking larger problems into smaller, manageable tasks. While working on my 3D scene, I used modular development strategies by creating separate functions for object transformations, texture loading, lighting, rendering, and camera controls. Iteration played a major role throughout development because each milestone built upon the previous one. As I progressed through the course, my coding approach became more organized and structured, allowing me to troubleshoot problems more effectively and create reusable code.

Computer science supports my goals by providing problem-solving skills and technical knowledge that can be applied across many industries. Computational graphics and visualizations introduced me to concepts such as 3D modeling, transformations, texture mapping, lighting, camera navigation, and rendering pipelines. These skills will benefit my future educational pathway as I continue studying software engineering and computer engineering concepts. Professionally, the knowledge gained from computational graphics and visualizations can be applied to game development, simulations, data visualization, user interface design, virtual reality, and other fields that rely on interactive computer-generated environments.
